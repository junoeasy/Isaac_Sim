### 내적
내적은 두 벡터가 `얼마나 같은 방향을 보고 있는지` 측정하는 계산
공식
```
a · b = a_x * b_x + a_y * b_y
a · b = |a||b|cos(theta)
```
ex1)
```
a = [1,0]
b = [1,0]
a · b = a_x * b_x + a_y * b_y 
= 1*1 + 0*0 
= 1 + 0 = 1
# 둘이 같은 방향을 보고 있다.
```
ex2)
```
a = [1,0]
b = [0,1]
a · b = a_x * b_x + a_y * b_y 
= 1*0 + 0*1 
= 0 + 0 = 0
# 둘이 수직이다.
```
ex3)
```
a = [1,0]
b = [-1,0]
a · b = a_x * b_x + a_y * b_y 
= 1*(-1) + 0*0 
= -1 + 0 =-1
# 서로 반대 방향이다.
```

### 역행렬

숫자에서 1에 3을 곱해서 3을 만들었을 떄 다시 돌리려면 3의 역수인 1/3을 곱하면 되는 것처럼

# @는 행렬 곱하기
```

### 동치좌표
좌표 뒤에 `1`을 붙인 표현
ex)
```
# 2D 점
p = [x,y]

#동치좌표
p = [x,y,1]

# 3D 점
p = [x,y,z]

#동치좌표
p = [x,y,z,1]
```
1을 붙이는 이유
- 회전과 이동을 행렬 하나로 같이 처리하기 위해서
- 3D 공간에서 어떤 점을 이동시키고 회전시키는 연산이 많이 발생하는데 회전은 행렬 곲(3X3), 이동은 벡터 더하기(3X1)로 연산 포맷이 다른데, 동차좌표를 써서 (4X1)로 늘리면 4x4로 합칠수 있게 되어 계산이 편해짐

### 2D 회전행렬 (2X2)
2차원 평면에서 회전은 기준축이 1개 밖에 없음. 
원점을 기준으로 반시계 방향으로  theta 만큼 회전시키는 행렬
  $$R_{2D}(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix}$$
기본 단위 벡터인 $\begin{bmatrix} 1 \\ 0 \end{bmatrix}$ 를 $\theta$ 만큼 회전시키면 삼각함수 정의에 의해 $\begin{bmatrix} \cos\theta \\ sin\theta \end{bmatrix}$ 가 된다.
마찬가지로 $\begin{bmatrix} 0 \\ 1 \end{bmatrix}$ 를 회전시키면 $\begin{bmatrix} -\sin\theta \\ \cos\theta \end{bmatrix}$ 가 된다.
2D 회전행렬을 $\begin{bmatrix} a & b \\ c & d \end{bmatrix}$ 라고 할 때 $\begin{bmatrix} a & b \\ c & d \end{bmatrix} \begin{bmatrix} 1 \\ 0 \end{bmatrix}=\begin{bmatrix} a \\  c \end{bmatrix}$ 이고, $\begin{bmatrix} a & b \\ c & d \end{bmatrix} \begin{bmatrix} 0 \\ 1 \end{bmatrix}=\begin{bmatrix} b \\  d \end{bmatrix}$ 이므로
$\begin{bmatrix} a \\ c \end{bmatrix}=\begin{bmatrix} \cos\theta \\ sin\theta \end{bmatrix}, \begin{bmatrix} b \\ d \end{bmatrix}= \begin{bmatrix} -\sin\theta \\ \cos\theta \end{bmatrix} , R_{2D}(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix}$ 가 된다. 

### 3D 회전행렬 (3X3)
3차원은 회전 축이 3개이므로 3개의 기준 축마다 회전 행렬이 다르다.
- X축 기준 회전
	- X축 좌표는 고정된 채  Y-Z 평면상 회전
	- $$R_x(\theta) = \begin{bmatrix} 1 & 0 & 0 \\ 0 & \cos\theta & -\sin\theta \\ 0 & \sin\theta & \cos\theta \end{bmatrix}$$
- Y축 기준 회전
	- Y축 좌표는 고정된 채 X-Z 평면 회전 (주의 : 삼각함수 부호의 위치가 대칭적으로 바뀜)
	- $$R_y(\theta) = \begin{bmatrix} \cos\theta & 0 & \sin\theta \\ 0 & 1 & 0 \\ -\sin\theta & 0 & \cos\theta \end{bmatrix}$$
- Z축 기준 회전
	- Z축 좌표는 고정된 채 X-Y 평면 회전
	- $$R_z(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta & 0 \\ \sin\theta & \cos\theta & 0 \\ 0 & 0 & 1 \end{bmatrix}$$
### Euler angle
3D 회전을 사람이 이해하기 쉽게 3 각도로 표현한 것
```
roll = x축 기준 회전
pitch = y축 기준 회전
yaw = z축 기준 회전
```

```
# 장점
사람이 읽기에는 좋다.

실제 3D 로봇 계산에서는 quaternion이나 회전행렬을 더 많이 사용
gimbal lock 문제가 생길 수도 있다.
- gimbal lock : Euler anglefh 3D 회전을 표현할 떄, 특정 자세에서 회전축 2개가 겹쳐 자유도 하나를 잃는 문제
  roll,pitch,yaw로 표현 되지만, 이전 회전에 따라 각 회전 축이 바뀔 수 있다 
  ex) yaw를 90도 했다는 가정하에 yaw이전 roll과 yaw 이후 roll 이 같은 축이다.

```
q = [x,y,z,w] 또는 [w,x,y,z]
```
사람이 직접 읽기는 어렵지만, 장점이 크다.
