# Link 분류 기준

## 기본 원칙

로봇의 link는 "이전 joint와 다음 joint 사이의 단단한 강체 덩어리"이다.

```text
base_link = world와 joint1 사이
link1     = joint1과 joint2 사이
link2     = joint2와 joint3 사이
link3     = joint3와 joint4 사이
link4     = joint4와 joint5 사이
link5     = joint5와 joint6 사이
link6     = joint6 이후 끝단
```

joint1을 움직이면 link1~link6이 모두 움직이는 것이 맞다. 하지만 link1은 "joint1 이후 전체"가 아니라 "joint1과 joint2 사이의 단단한 부분"이다.

## 서보모터 기준

각 서보마다 다음 기준으로 분류한다.

```text
서보 케이스 / 몸통 / 고정 브라켓
-> 이전 link

서보 horn / 출력축에 붙은 부품 / 다음 팔을 돌리는 부품
-> 다음 link
```

예:

```text
joint1 servo case -> base_link
joint1 horn       -> link1

joint2 servo case -> link1
joint2 horn       -> link2
```

## 볼트/너트 기준

볼트와 너트는 처음부터 물리용 rigid body로 만들지 않는다.

```text
visual: 유지 가능
collision: 보통 끔
rigid body: 주지 않음
mass: 해당 link 질량에 포함한다고 생각
```

처음 테스트에서는 볼트/너트가 꼭 링크에 들어가 있지 않아도 된다. 큰 부품과 joint가 먼저 맞아야 한다.

## 현재 수정된 중요 부품

`tn___1_XF5i1JPa1g4X18n0`는 `base_link`에 있어야 한다.

```text
기존: link1/tn___1_XF5i1JPa1g4X18n0
변경: base_link/tn___1_XF5i1JPa1g4X18n0
```

`tn__U_n9Bm0n0Z4m0`는 joint1 쪽, 즉 현재 구조에서는 `link1`에 있어야 한다.

```text
기존: link2/tn__U_n9Bm0n0Z4m0
변경: link1/tn__U_n9Bm0n0Z4m0
```
