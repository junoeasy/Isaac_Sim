# 에러와 해결 노트

## Cannot move/rename ancestral prim

메시지:

```text
Cannot move/rename ancestral prim
```

원인:

CAD 변환 USD가 instance/prototype/reference 구조로 되어 있어, 해당 prim을 직접 이동할 수 없었다.

해결:

`6 dof arm_editable_flat.usd`를 만들어 instanceable 구조를 풀었다.

```text
/home/junho/isaac/6 dof arm_editable_flat.usd
```

## USD를 드래그하면 보이는데 직접 열면 안 보임

원인:

USD를 Stage로 드래그하면 `/World` 아래 reference로 붙어서 보인다. 하지만 이 경우 자식 prim 이동이 막힐 수 있다.

직접 `File > Open`으로 열면 CAD 루트만 열리고 `/World`, 카메라, 조명이 없어서 안 보이는 것처럼 보일 수 있다.

해결:

```text
File > Open으로 직접 열기
최상위 prim 선택
F 키로 Frame Selected
```

## Articulations with kinematic bodies are not supported

메시지:

```text
Articulations with kinematic bodies are not supported,
please exclude joint (...) from articulation.
```

원인:

`base_link`를 고정하려고 `Kinematic Enabled`를 켰는데, PhysX articulation 내부에는 kinematic rigid body가 들어갈 수 없다.

해결:

`base_link`의 kinematic을 끄고, 대신 `world_fixed_base` Fixed Joint로 월드에 고정했다.

현재 해결된 파일:

```text
/home/junho/isaac/6 dof arm_joint2_test_parent_to_base_v6_fixed_base.usd
```

## Play 누르면 로봇이 아래로 떨어짐

원인:

base도 dynamic rigid body이고 ground나 world 고정이 없으면 중력 때문에 전체가 떨어진다.

해결:

```text
base_link를 kinematic으로 만들지 말고
world_fixed_base Fixed Joint로 월드에 고정
```

## joint target이 정확히 90도까지 안 가고 밀림

원인:

Joint drive는 이상적인 위치 고정이 아니라 토크 제한이 있는 PD 제어기이다. 다른 링크의 무게, 관성, joint2 동작 때문에 joint1이 목표 각도에 완전히 도달하지 못할 수 있다.

해결:

```text
Max Force 크게
Stiffness 크게
Damping 충분히 크게
필요하면 테스트 중 joint limit을 좁게 설정
```

예:

```text
Stiffness: 10000
Damping: 3000
Max Force: 1000000
```

joint2만 테스트할 때 joint1을 고정하고 싶으면:

```text
joint1 lower limit = 90
joint1 upper limit = 90
```

또는:

```text
joint1 lower limit = 89.9
joint1 upper limit = 90.1
```
